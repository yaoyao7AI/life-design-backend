# 🔧 数据库连接错误解决方案

## ❌ 错误日志分析

**从日志看到的问题：**
```
Aborted connection XXXX to db: 'unconnected' user: 'unauthenticated' host: '100.104.72.7' (Got an error reading communication packets)
Got an error reading communication packets
```

**问题分析：**
- ❌ 连接被中止（Aborted connection）
- ❌ 用户未认证（unauthenticated）
- ❌ 通信包读取错误（Got an error reading communication packets）

---

## 🚨 可能的原因

### 原因 1：白名单未配置（最可能）

**问题：**
- 后端服务器的 IP 地址不在数据库白名单中
- 数据库拒绝连接

**解决：**
- 添加后端服务器 IP 到白名单

---

### 原因 2：数据库用户名或密码错误

**问题：**
- 环境变量中的数据库用户名或密码错误
- 认证失败

**解决：**
- 检查并更新环境变量

---

### 原因 3：数据库连接地址错误

**问题：**
- 环境变量中的数据库主机地址错误
- 无法连接到数据库

**解决：**
- 检查并更新数据库连接地址

---

## ✅ 解决方案

### 步骤 1：添加白名单（必须）

**在阿里云 RDS 控制台：**

1. **进入实例详情页**
   - 实例ID：`rm-2ze0yppdih8t4t4e4`
   - 点击左侧菜单："白名单与安全组"

2. **添加后端服务器 IP**
   - 点击"添加白名单分组"或"修改"
   - 添加后端服务器 IP：`123.56.17.118`
   - 或添加 `0.0.0.0/0`（允许所有 IP，仅用于测试）

3. **保存配置**

---

### 步骤 2：检查数据库连接信息

**需要确认的信息：**

1. **数据库主机地址：**
   - 在 RDS 控制台 → 数据库连接 → 查看"内网地址"
   - 格式：`rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com`

2. **数据库端口：**
   - 通常是 `3306`

3. **数据库名称：**
   - `life_design`

4. **数据库用户名和密码：**
   - 你之前创建的数据库账号

---

### 步骤 3：更新后端环境变量

**在服务器上更新 `.env` 文件：**

```bash
# SSH 连接到服务器
ssh -i ~/Desktop/ssh-keys/life-design-key.pem root@123.56.17.118

# 编辑环境变量文件
cd /root/life-design-backend
nano .env
```

**更新以下配置：**
```env
DB_HOST=rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com
DB_PORT=3306
DB_USER=你的数据库用户名
DB_PASS=你的数据库密码
DB_NAME=life_design
```

**保存并重启后端：**
```bash
pm2 restart life-design-backend
```
