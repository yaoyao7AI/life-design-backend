# 📝 创建数据库表单填写指南

## 🎯 目标
通过阿里云 RDS 图形界面创建 `life_design` 数据库

---

## 📋 表单填写步骤

### 1. 数据库 (DB) 名称（必填）

**输入框：** "* 数据库 (DB) 名称"

**填写：**
```
life_design
```

**规则：**
- ✅ 只能包含：小写字母、数字、下划线、中划线
- ✅ 必须以小写字母开头
- ✅ 必须以小写字母或数字结尾
- ✅ 最多64个字符

**示例：**
- ✅ `life_design`（推荐）
- ✅ `life-design`（也可以）
- ✅ `lifedesign`（也可以）
- ❌ `LifeDesign`（不能有大写字母）
- ❌ `123life`（不能以数字开头）

---

### 2. 支持字符集（必填）

**下拉框：** "* 支持字符集"

**选择：**
```
utf8mb4
```

**操作：**
1. **点击下拉框**
2. **选择 `utf8mb4`**
   - 如果下拉框显示的是 `utf8`，点击后选择 `utf8mb4`
   - `utf8mb4` 支持完整的 UTF-8 字符集（包括 emoji）

**为什么选择 `utf8mb4`：**
- ✅ 支持完整的 Unicode 字符
- ✅ 支持 emoji 表情
- ✅ 是 MySQL 推荐的字符集

---

### 3. 授权账号（可选）

**下拉框：** "授权账号"

**选择：**
```
选择你刚创建的账号
```

**操作：**
1. **点击下拉框**
2. **选择你之前创建的数据库账号**
   - 如果看到你创建的账号（如 `life_design_user`），选择它
   - 如果列表为空或只有"未授权账号 (默认)"，可以：
     - 选择"未授权账号 (默认)"（稍后可以授权）
     - 或点击"创建新账号"创建新账号

**说明：**
- 授权账号可以让该账号访问这个数据库
- 如果暂时不授权，稍后可以在"账号管理"中授权

---

### 4. 备注说明（可选）

**文本域：** "备注说明"

**可以填写：**
```
用于生活设计应用的用户数据存储
```

**或留空：**
- 这个字段是可选的
- 可以留空不填

**如果使用 AI 生成：**
- 点击"AI生成"按钮（紫色，带星星图标）
- AI 会自动生成备注（可选）

## ✅ 创建成功后的下一步

**数据库创建成功后，需要：**

### 1. 切换到 DMS SQL 控制台

1. **访问 DMS：** https://dms.console.aliyun.com
2. **选择实例：** `rm-2ze0yppdih8t4t4e4`
3. **选择数据库：** `life_design`

---

### 2. 创建 users 表

**在 DMS SQL 控制台执行：**

```sql
USE life_design;

CREATE TABLE users (
  id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '用户ID',
  phone VARCHAR(20) NOT NULL UNIQUE COMMENT '手机号',
  password VARCHAR(255) NULL COMMENT '密码',
  nickname VARCHAR(50) COMMENT '昵称',
  avatar VARCHAR(255) COMMENT '头像URL',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  INDEX idx_phone (phone)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';

DESCRIBE users;
```

**点击"执行(F8)"**

## 🚨 常见问题

### 问题 1：数据库名称不符合规则

**错误提示：** "数据库名称格式不正确"

**解决：**
- 确保只使用小写字母、数字、下划线、中划线
- 必须以小写字母开头
- 必须以小写字母或数字结尾

---

### 问题 2：找不到 utf8mb4 字符集

**如果下拉框没有 `utf8mb4`：**

**选择：** `utf8`（也可以，但推荐 utf8mb4）

**或者：**
- 创建数据库后，在 DMS 中执行：
  ```sql
  ALTER DATABASE life_design CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
  ```

---

### 问题 3：创建失败

**可能原因：**
- 数据库名称已存在
- 权限不足
- 网络问题

**解决：**
- 更换数据库名称（如：`life_design_new`）
- 检查权限
- 刷新页面重试
