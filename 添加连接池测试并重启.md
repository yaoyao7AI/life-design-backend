# 🔧 添加连接池测试并重启

## ❌ 问题分析

**现象：**
- ✅ 测试时单个连接成功
- ✅ 测试时连接池连接成功
- ❌ 但应用运行时仍然报 `ETIMEDOUT`

**可能的原因：**
- 连接池在创建时没有立即连接
- 第一次查询时才尝试连接，此时可能超时
- 需要在连接池创建后立即测试连接

---

## 🔧 已添加的修复

**我已经更新了 `src/db.js`，在连接池创建后立即测试连接：**

```javascript
// 在连接池创建后立即测试连接
pool.getConnection()
  .then(connection => {
    console.log('[DB] ✅ 连接池连接测试成功');
    connection.release();
  })
  .catch(error => {
    console.error('[DB] ❌ 连接池连接测试失败:', error.code, error.message);
  });
```

**这样可以在启动时就发现连接问题，而不是等到第一次查询时。**

## 🚀 部署步骤

### 在服务器上执行：

```bash
cd /root/apps/life-design-backend

# 备份原文件
cp src/db.js src/db.js.backup.$(date +%Y%m%d_%H%M%S)

# 更新文件（添加连接池测试）
cat > src/db.js << 'EOF'
import mysql from "mysql2/promise";
import dotenv from "dotenv";

dotenv.config();

console.log('[DB] 创建连接池，DB_HOST:', process.env.DB_HOST);
console.log('[DB] DB_PORT:', process.env.DB_PORT);
console.log('[DB] DB_USER:', process.env.DB_USER);
console.log('[DB] DB_NAME:', process.env.DB_NAME);

export const pool = mysql.createPool({
  host: process.env.DB_HOST,
  port: Number(process.env.DB_PORT),
  user: process.env.DB_USER,
  password: process.env.DB_PASS,
  database: process.env.DB_NAME,
  connectionLimit: 10,
  connectTimeout: 30000,
  enableKeepAlive: true,
  keepAliveInitialDelay: 0,
});

// 在连接池创建后立即测试连接
pool.getConnection()
  .then(connection => {
    console.log('[DB] ✅ 连接池连接测试成功');
    connection.release();
  })
  .catch(error => {
    console.error('[DB] ❌ 连接池连接测试失败:', error.code, error.message);
  });
EOF

# 验证配置
echo "=== 验证更新后的配置 ==="
cat src/db.js | tail -10

# 重启服务
echo ""
echo "=== 重启服务 ==="
pm2 stop life-design-backend
pm2 delete life-design-backend
pm2 save --force
sleep 3
pm2 start src/server.js --name life-design-backend --update-env
sleep 5
pm2 logs life-design-backend --lines 40 --nostream
```
