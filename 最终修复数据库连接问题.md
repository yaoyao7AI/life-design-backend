## ✅ 解决方案

### 方案 1：如果环境变量名称不匹配

**检查 `src/routes/auth.js` 使用的数据库文件：**
- 如果使用 `src/db.js` → 需要 `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASS`, `DB_NAME`
- 如果使用 `src/config/database.js` → 需要 `MYSQL_HOST`, `MYSQL_PORT`, `MYSQL_USER`, `MYSQL_PASSWORD`, `MYSQL_DATABASE`

**更新 `.env` 文件，确保环境变量名称匹配！**

---

### 方案 2：完全重置环境变量

**在服务器上执行：**

```bash
cd /root/apps/life-design-backend

# 备份
cp .env .env.backup.$(date +%Y%m%d_%H%M%S)

# 更新配置（包含所有可能的环境变量名称）
cat > .env << 'EOF'
# 数据库配置（src/db.js 使用）
DB_HOST=rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com
DB_PORT=3306
DB_USER=life_design_user
DB_PASS=YOUR_DB_PASSWORD
DB_NAME=life_design

# 数据库配置（src/config/database.js 使用，如果存在）
MYSQL_HOST=rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com
MYSQL_PORT=3306
MYSQL_USER=life_design_user
MYSQL_PASSWORD=YOUR_DB_PASSWORD
MYSQL_DATABASE=life_design

# 其他配置
JWT_SECRET=your_secret_key_change_this_in_production
ALIYUN_ACCESS_KEY_ID=YOUR_ALIYUN_ACCESS_KEY_ID
ALIYUN_ACCESS_KEY_SECRET=YOUR_ALIYUN_ACCESS_KEY_SECRET

# 完全重启
pm2 delete life-design-backend
```
