# 🔧 修复数据库配置错误

## ❌ 错误分析

**从日志看到：**
```
Error: getaddrinfo ENOTFOUND rm-2zec076upfs3zd44110.mysql.rds.aliyuncs.com
```

**问题：**
- 数据库主机地址错误：`rm-2zec076upfs3zd44110.mysql.rds.aliyuncs.com`
- 正确的地址应该是：`rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com`

**原因：**
- `.env` 文件可能没有正确更新
- 或者有多个 `.env` 文件

---

## ✅ 解决方案

### 步骤 1：检查当前配置

**在服务器上执行：**

```bash
cd /root/life-design-backend
cat .env
```

**查看 `DB_HOST` 的值是否正确。**

---

### 步骤 2：更新配置

**如果配置不正确，执行：**

```bash
cd /root/life-design-backend

# 备份现有配置
cp .env .env.backup.$(date +%Y%m%d_%H%M%S)

# 更新配置
cat > .env << 'EOF'
DB_HOST=rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com
DB_PORT=3306
DB_USER=life_design_user
DB_PASS=YOUR_DB_PASSWORD
DB_NAME=life_design
EOF

# 验证配置
cat .env
```

---

### 步骤 3：重启后端服务

```bash
pm2 restart life-design-backend
```

---

### 步骤 4：查看日志确认

```bash
pm2 logs life-design-backend --lines 20
```

**应该看到：**
- ✅ 服务启动成功
- ✅ 没有 `ENOTFOUND` 错误
- ✅ 数据库连接成功

---

## 🎯 快速修复命令

**复制以下全部命令，一次性执行：**

```bash
cd /root/life-design-backend && \
cp .env .env.backup.$(date +%Y%m%d_%H%M%S) && \
cat > .env << 'EOF'
DB_HOST=rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com
DB_PORT=3306
DB_USER=life_design_user
DB_PASS=YOUR_DB_PASSWORD
DB_NAME=life_design
EOF
echo "✅ 配置已更新" && \
cat .env && \
echo "" && \
pm2 restart life-design-backend && \
sleep 3 && \
pm2 logs life-design-backend --lines 20
```
