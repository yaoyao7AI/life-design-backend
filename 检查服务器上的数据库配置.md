# 🔍 检查服务器上的数据库配置

## ❌ 问题

**代码中有两个数据库配置文件：**
1. `src/db.js` - 使用 `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASS`, `DB_NAME`
2. `src/config/database.js` - 使用 `MYSQL_HOST`, `MYSQL_PORT`, `MYSQL_USER`, `MYSQL_PASSWORD`, `MYSQL_DATABASE`

**需要检查服务器上实际使用的是哪个文件！**

---

## 🔍 检查步骤

### 步骤 1：检查服务器上的代码

**在服务器上执行：**

```bash
cd /root/apps/life-design-backend

# 检查数据库配置文件
echo "=== src/db.js ==="
cat src/db.js

echo ""
echo "=== src/config/database.js ==="
cat src/config/database.js 2>/dev/null || echo "文件不存在"

echo ""
echo "=== 检查哪个文件被导入 ==="
grep -r "from.*db\|require.*db\|from.*database\|require.*database" src/routes/auth.js
```

---

### 步骤 2：检查环境变量名称

**如果使用的是 `src/config/database.js`，需要更新环境变量名称：**

```bash
cd /root/apps/life-design-backend

# 检查 .env 文件
cat .env | grep -E "DB_|MYSQL_"
```

---

## ✅ 解决方案

### 方案 1：如果使用的是 `src/db.js`

**环境变量名称正确，但需要确认已更新：**

```bash
cd /root/apps/life-design-backend
cat .env | grep DB_HOST
```

---

### 方案 2：如果使用的是 `src/config/database.js`

**需要更新环境变量名称：**

```bash
cd /root/apps/life-design-backend

# 备份
cp .env .env.backup.$(date +%Y%m%d_%H%M%S)

# 更新环境变量名称
cat >> .env << 'EOF'
MYSQL_HOST=rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com
MYSQL_PORT=3306
MYSQL_USER=life_design_user
MYSQL_PASSWORD=YOUR_DB_PASSWORD
MYSQL_DATABASE=life_design
EOF

# 重启服务
pm2 restart life-design-backend --update-env
```
