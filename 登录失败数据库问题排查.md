# 🔍 登录失败 - 数据库连接问题排查

## ❌ 问题确认

**测试结果：**
- ✅ 发送验证码成功
- ❌ 登录失败：返回 `{"error":"登录失败"}`

**问题分析：**
- 验证码验证可能通过（否则会返回"验证码错误"）
- 数据库操作失败（catch 块返回"登录失败"）

---

## 🚨 可能的原因

### 原因 1：后端未连接到新数据库（最可能）

**问题：**
- 后端仍连接到旧数据库或错误的数据库
- 新数据库 `life_design` 在实例 `rm-2ze0yppdih8t4t4e4` 下
- 后端环境变量可能还是旧配置

**解决：**
- 更新后端环境变量配置
- 确认连接到新数据库

---

### 原因 2：数据库连接配置错误

**问题：**
- 环境变量配置错误
- 数据库主机地址错误
- 数据库名称错误

**解决：**
- 检查环境变量配置
- 更新为新数据库的连接信息

---

## 🔧 排查步骤

### 步骤 1：检查后端环境变量

**在服务器上检查：**

```bash
# SSH 连接到服务器
ssh -i ~/Desktop/ssh-keys/life-design-key.pem root@123.56.17.118

# 检查环境变量
cd /root/life-design-backend
cat .env | grep DB_
```

**应该看到：**
```
DB_HOST=rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com
DB_PORT=3306
DB_USER=你的数据库用户名
DB_PASS=你的数据库密码
DB_NAME=life_design
```

---

### 步骤 2：检查后端日志

**查看后端日志，找到具体错误：**

```bash
# 在服务器上
pm2 logs life-design-backend --lines 50
```

**查找：**
- `[登录错误]` 日志
- 数据库连接错误
- SQL 错误信息

---

### 步骤 3：测试数据库连接

**在服务器上测试：**

```bash
# 测试数据库连接
mysql -h rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com -u 用户名 -p life_design
```

---

## ✅ 解决方案

### 方案 1：更新后端环境变量（推荐）

**在服务器上更新 `.env` 文件：**

```bash
# SSH 连接到服务器
ssh -i ~/Desktop/ssh-keys/life-design-key.pem root@123.56.17.118

# 编辑环境变量文件
cd /root/life-design-backend
nano .env
```

**更新以下配置：**
```env
DB_HOST=rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com
DB_PORT=3306
DB_USER=你的数据库用户名
DB_PASS=你的数据库密码
DB_NAME=life_design
```

**重启后端：**
```bash
pm2 restart life-design-backend
```

---

### 方案 2：检查数据库连接信息

**需要确认的信息：**
1. **数据库主机地址：** `rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com`
2. **数据库端口：** `3306`
3. **数据库名称：** `life_design`
4. **数据库用户名：** （你创建的账号）
5. **数据库密码：** （你设置的密码）

**在阿里云 RDS 控制台查看：**
- 实例详情 → 连接信息
- 可以看到数据库主机地址和端口
