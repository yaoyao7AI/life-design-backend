# 🔍 检查代码中的硬编码数据库地址

## ❌ 问题

**错误日志仍然显示旧的数据库地址：**
```
Error: getaddrinfo ENOTFOUND rm-2zec076upfs3zd44110.mysql.rds.aliyuncs.com
```

**但配置文件中已经是正确的：**
```
DB_HOST=rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com
```

**说明代码中可能有硬编码的数据库地址！**

---

## 🔍 检查步骤

### 步骤 1：搜索代码中的硬编码地址

**在服务器上执行：**

```bash
cd /root/apps/life-design-backend

# 搜索旧的数据库地址
grep -r "rm-2zec076upfs3zd44" src/
grep -r "rm-2zec076upfs3zd44110" src/
grep -r "2zec076upfs3zd44" src/

# 检查数据库配置文件
cat src/db.js
cat src/config/database.js 2>/dev/null || echo "文件不存在"
```

---

### 步骤 2：检查环境变量加载

**在服务器上执行：**

```bash
cd /root/apps/life-design-backend

# 检查代码如何读取环境变量
grep -r "DB_HOST" src/
grep -r "process.env" src/ | grep -i db
```

---

## ✅ 解决方案

### 如果代码中有硬编码地址

**需要修改代码，使用环境变量：**

```javascript
// 错误示例（硬编码）
const host = 'rm-2zec076upfs3zd44110.mysql.rds.aliyuncs.com';

// 正确示例（使用环境变量）
const host = process.env.DB_HOST;
```

---

### 如果环境变量没有正确加载

**检查代码中的环境变量读取：**

```javascript
// 确保正确加载 dotenv
import dotenv from 'dotenv';
dotenv.config();

// 或者
require('dotenv').config();
```

---

## 🎯 快速检查命令

**复制以下全部命令，一次性执行：**

```bash
cd /root/apps/life-design-backend && \
echo "=== 搜索硬编码地址 ===" && \
grep -r "rm-2zec076upfs3zd44" src/ 2>/dev/null || echo "未找到硬编码地址" && \
echo "" && \
echo "=== 检查数据库配置文件 ===" && \
cat src/db.js && \
echo "" && \
echo "=== 检查环境变量使用 ===" && \
grep -r "DB_HOST\|process.env" src/ | head -10
```
