# 🗑️ 清空旧数据并完成迁移

## 🎯 目标
1. 添加 `password` 字段到 `users` 表
2. 清空 `users` 表中的旧数据（只保留表结构）

---

## ⚠️ 重要提醒

**清空数据是不可逆操作！**

- ❌ **删除后无法恢复**
- ✅ **只删除数据，保留表结构**
- ✅ **表结构（字段）会保留**

---

## 📋 执行步骤（2步）

### 步骤 1：添加 `password` 字段（如果还没添加）

**在 DMS SQL 控制台执行：**

```sql
USE life_design;
ALTER TABLE users ADD COLUMN password VARCHAR(255) NULL AFTER phone;
```

**点击"执行(F8)"**

**如果字段已存在，会提示错误，可以忽略。**

---

### 步骤 2：清空 `users` 表数据

**⚠️ 警告：这会删除所有用户数据！**

**在 DMS SQL 控制台执行：**

```sql
USE life_design;
TRUNCATE TABLE users;
```

**或者使用 DELETE（效果相同）：**

```sql
USE life_design;
DELETE FROM users;
```

**点击"执行(F8)"**

---

## 🔍 两种清空方式的区别

### TRUNCATE TABLE（推荐）

```sql
TRUNCATE TABLE users;
```

**特点：**
- ✅ 速度快
- ✅ 重置自增ID（下次插入从1开始）
- ✅ 不记录日志（无法回滚）
- ✅ 释放表空间

---

### DELETE FROM

```sql
DELETE FROM users;
```

**特点：**
- ⚠️ 速度较慢（数据多时）
- ⚠️ 保留自增ID（下次插入继续递增）
- ✅ 记录日志（可以回滚）
- ⚠️ 不释放表空间

**推荐使用 `TRUNCATE TABLE`**

---

## ✅ 完整 SQL（一键执行）

**复制以下全部内容，一次性执行：**

```sql
USE life_design;

-- 1. 添加 password 字段（如果不存在）
ALTER TABLE users ADD COLUMN password VARCHAR(255) NULL AFTER phone;

-- 2. 清空所有数据
TRUNCATE TABLE users;

-- 3. 验证表结构（确认字段存在）
DESCRIBE users;

-- 4. 验证数据已清空（应该返回空结果）
SELECT COUNT(*) FROM users;
```

**点击"执行(F8)"**

---

## 🚨 注意事项

### 1. 备份提醒

**如果以后可能需要恢复数据：**

**清空前先备份：**
```sql
-- 导出数据（在DMS中可以导出）
SELECT * FROM users INTO OUTFILE '/tmp/users_backup.csv';
```

---

### 2. 相关数据

**如果 `users` 表与其他表有关联：**

- ⚠️ 清空 `users` 表可能影响其他表的外键关系
- ⚠️ 如果有外键约束，可能需要先删除关联数据

**检查外键关系：**
```sql
SHOW CREATE TABLE users;
```

---

### 3. 自增ID重置

**使用 `TRUNCATE TABLE` 后：**
- ✅ 自增ID会重置为1
- ✅ 下次插入数据时，ID从1开始

**使用 `DELETE FROM` 后：**
- ⚠️ 自增ID不会重置
- ⚠️ 下次插入数据时，ID继续递增
