# 🔍 检查并修复配置

## ❌ 问题

**错误日志显示：**
```
Error: getaddrinfo ENOTFOUND rm-2zec076upfs3zd4411o.mysql.rds.aliyuncs.com
```

**说明配置可能没有正确更新。**

---

## 🔧 解决步骤

### 步骤 1：检查当前配置

**在服务器上执行：**

```bash
cd /root/life-design-backend
cat .env
```

**查看 `DB_HOST` 的值是什么。**

---

### 步骤 2：如果配置不对，重新更新

**执行以下命令：**

```bash
cd /root/life-design-backend

# 查看当前配置
echo "=== 当前配置 ==="
cat .env
echo ""

# 备份
cp .env .env.backup.$(date +%Y%m%d_%H%M%S)

# 更新配置
cat > .env << 'EOF'
DB_HOST=rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com
DB_PORT=3306
DB_USER=life_design_user
DB_PASS=YOUR_DB_PASSWORD
DB_NAME=life_design
EOF

# 验证配置
echo "=== 更新后的配置 ==="
cat .env
echo ""

# 重启服务
pm2 restart life-design-backend

# 等待并查看日志
sleep 5
pm2 logs life-design-backend --lines 30 --nostream
```

---

### 步骤 3：检查是否有其他配置文件

**检查是否有多个 .env 文件：**

```bash
cd /root/life-design-backend
find . -name ".env*" -type f
```

**检查项目目录是否正确：**

```bash
pwd
ls -la
```
