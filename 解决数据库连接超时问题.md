# 🔧 解决数据库连接超时（ETIMEDOUT）问题

## ❌ 问题分析

**错误：`ETIMEDOUT`（连接超时）**

**说明：**
- ✅ DNS 解析成功（不再是 `ENOTFOUND`）
- ❌ 连接被阻止或超时

**可能的原因：**
1. **白名单未配置**（最可能）⚠️
   - 后端服务器 IP 不在 RDS 白名单中
2. **使用了外网地址但未开通外网访问**
   - 应该使用内网地址（如果服务器和 RDS 在同一 VPC）
3. **网络不通**
   - VPC 配置问题

---

## 🔍 排查步骤

### 步骤 1：确认后端服务器 IP 地址

**在服务器上执行：**

```bash
# 查看服务器公网 IP
curl -s ifconfig.me

# 查看服务器内网 IP
hostname -I | awk '{print $1}'
```

**记录这两个 IP 地址！**

---

### 步骤 2：检查 RDS 白名单配置

**在阿里云 RDS 控制台：**

1. 进入 **RDS 控制台**
2. 找到实例：`rm-2ze0yppdih8t4t4e4`
3. 点击 **数据安全性** → **白名单设置**
4. 检查是否包含后端服务器 IP：
   - **公网 IP**：`123.56.17.118`（如果使用外网地址）
   - **内网 IP**：（如果使用内网地址）

---

### 步骤 3：检查是否应该使用内网地址

**如果后端服务器和 RDS 在同一 VPC：**
- ✅ **应该使用内网地址**（更快、更安全、免费）
- ❌ **不应该使用外网地址**（慢、需要开通外网、可能收费）

**如何判断：**
- 查看 RDS 实例的 **连接信息**
- 如果有 **内网地址**，优先使用内网地址
- 内网地址格式：`rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com`（可能相同）
- 或者：`rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com`（内网）

---

## ✅ 解决方案

### 方案 1：添加白名单（如果使用外网地址）

**在阿里云 RDS 控制台：**

1. 进入 **RDS 控制台**
2. 找到实例：`rm-2ze0yppdih8t4t4e4`
3. 点击 **数据安全性** → **白名单设置**
4. 点击 **修改**
5. 在 **白名单设置** 中添加：
   - `123.56.17.118`（后端服务器公网 IP）
   - 或者添加 `0.0.0.0/0`（允许所有 IP，仅用于测试，不安全）
6. 点击 **确定**

---

### 方案 2：使用内网地址（推荐）

**如果后端服务器和 RDS 在同一 VPC：**

1. **获取内网地址：**
   - 在 RDS 控制台查看 **连接信息**
   - 找到 **内网地址**（格式可能相同，但需要确认）

2. **更新服务器配置：**

```bash
cd /root/apps/life-design-backend

# 备份配置
cp .env .env.backup.$(date +%Y%m%d_%H%M%S)

# 更新为内网地址（如果内网地址不同）
# 注意：如果内网地址和当前地址相同，跳过此步骤
cat > .env << 'EOF'
DB_HOST=rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com
DB_PORT=3306
DB_USER=life_design_user
DB_PASS=YOUR_DB_PASSWORD
DB_NAME=life_design
JWT_SECRET=your_secret_key_change_this_in_production
ALIYUN_ACCESS_KEY_ID=YOUR_ALIYUN_ACCESS_KEY_ID
ALIYUN_ACCESS_KEY_SECRET=YOUR_ALIYUN_ACCESS_KEY_SECRET
ALIYUN_SMS_SIGN=海口龙华陀恬雅百货商行
ALIYUN_SMS_TEMPLATE=SMS_499120253
PORT=3000
NODE_ENV=development
FRONTEND_BASE_URL=http://localhost:5174
EOF

# 重启服务
pm2 restart life-design-backend --update-env
```

---

### 方案 3：开通外网访问（如果必须使用外网）

**在阿里云 RDS 控制台：**

1. 进入 **RDS 控制台**
2. 找到实例：`rm-2ze0yppdih8t4t4e4`
3. 点击 **数据库连接**
4. 检查是否有 **外网地址**
5. 如果没有，点击 **申请外网地址**
6. **注意：** 开通外网可能需要额外费用

## 🎯 推荐操作流程

### 第一步：检查白名单（必须）

**在阿里云 RDS 控制台：**

1. 进入 **RDS 控制台**
2. 找到实例：`rm-2ze0yppdih8t4t4e4`
3. 点击 **数据安全性** → **白名单设置**
4. **添加后端服务器 IP：`123.56.17.118`**
5. 点击 **确定**

---

### 第二步：重新测试连接

**在服务器上执行：**

```bash
cd /root/apps/life-design-backend

node -e "
const mysql = require('mysql2/promise');
require('dotenv').config();

(async () => {
  try {
    console.log('=== 测试数据库连接 ===');
    console.log('DB_HOST:', process.env.DB_HOST);
    console.log('');
    
    const pool = mysql.createPool({
      host: process.env.DB_HOST,
      port: Number(process.env.DB_PORT),
      user: process.env.DB_USER,
      password: process.env.DB_PASS,
      database: process.env.DB_NAME,
      connectionLimit: 1,
      connectTimeout: 10000
    });
    
    console.log('正在连接数据库...');
    const connection = await pool.getConnection();
    console.log('✅ 数据库连接成功！');
    
    const [rows] = await connection.query('SELECT 1 as test');
    console.log('✅ 查询成功:', rows);
    
    connection.release();
    await pool.end();
    process.exit(0);
  } catch (error) {
    console.error('❌ 数据库连接失败:');
    console.error('错误代码:', error.code);
    console.error('错误信息:', error.message);
    process.exit(1);
  }
})();
"
```

---

### 第三步：如果仍然失败

**检查以下内容：**

1. **白名单是否添加成功？**
   - 确认 IP 地址正确
   - 确认保存成功

2. **是否应该使用内网地址？**
   - 检查 RDS 连接信息
   - 如果内网地址不同，更新配置

3. **外网是否开通？**
   - 如果使用外网地址，确认已开通外网访问
