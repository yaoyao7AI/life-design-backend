# 🔄 完全重启并检查代码

## ❌ 问题

**错误仍然显示 `PromisePool.query`，但代码应该已经更新为 `getConnection()`。**

**可能的原因：**
- PM2 缓存了旧代码
- 文件没有完全更新
- 需要完全重启 PM2

---

## 🔍 检查文件内容

**在服务器上执行：**

```bash
cd /root/apps/life-design-backend

# 检查第 75 行的实际内容
sed -n '75p' src/routes/auth.js

# 检查是否还有 pool.query
grep -n "pool\.query" src/routes/auth.js

# 检查是否有 getConnection
grep -n "getConnection" src/routes/auth.js
```

---

## 🔄 完全重启 PM2

**如果文件已更新，但 PM2 仍使用旧代码：**

```bash
cd /root/apps/life-design-backend

# 完全停止并删除进程
pm2 stop life-design-backend
pm2 delete life-design-backend
pm2 kill  # 停止 PM2 daemon
pm2 resurrect  # 重新启动（但这次会失败，因为我们删除了进程）

# 或者更简单的方法：
pm2 stop all
pm2 delete all
pm2 kill
pm2 save --force

# 重新启动
cd /root/apps/life-design-backend
pm2 start src/server.js --name life-design-backend --update-env

# 等待并查看日志
sleep 5
pm2 logs life-design-backend --lines 30 --nostream
```

---

## 🎯 一键检查和重启命令

**复制以下全部命令，一次性执行：**

```bash
cd /root/apps/life-design-backend && \
echo "=== 1. 检查文件内容 ===" && \
echo "第 75 行:" && \
sed -n '75p' src/routes/auth.js && \
echo "" && \
echo "是否有 pool.query:" && \
grep -n "pool\.query" src/routes/auth.js | head -3 || echo "✅ 没有找到 pool.query" && \
echo "" && \
echo "是否有 getConnection:" && \
grep -n "getConnection" src/routes/auth.js | head -3 && \
echo "" && \
echo "=== 2. 完全重启 PM2 ===" && \
pm2 stop life-design-backend 2>/dev/null || true && \
pm2 delete life-design-backend 2>/dev/null || true && \
pm2 save --force && \
sleep 3 && \
pm2 start src/server.js --name life-design-backend --update-env && \
sleep 5 && \
echo "" && \
echo "=== 3. 查看日志 ===" && \
pm2 logs life-design-backend --lines 30 --nostream
```
