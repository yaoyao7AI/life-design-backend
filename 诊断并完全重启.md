# 🔍 诊断并完全重启

## ❌ 问题

**即使更新了配置，仍然报 `ETIMEDOUT` 错误。**

**可能的原因：**
1. 连接池在模块加载时就创建了，使用了旧的环境变量
2. PM2 进程缓存了环境变量
3. 需要完全停止并重新启动，确保连接池重新创建

---

## 🔍 诊断步骤

### 在服务器上执行：

```bash
cd /root/apps/life-design-backend

# 1. 检查 .env 文件内容
echo "=== 1. 检查 .env 文件 ==="
cat .env | grep DB_HOST

# 2. 检查 PM2 进程的环境变量
echo ""
echo "=== 2. 检查 PM2 环境变量 ==="
pm2 env 0 | grep DB_HOST || echo "无法获取环境变量"

# 3. 检查代码中实际使用的配置
echo ""
echo "=== 3. 检查代码配置 ==="
node -e "
require('dotenv').config();
console.log('DB_HOST:', process.env.DB_HOST);
console.log('DB_PORT:', process.env.DB_PORT);
console.log('DB_USER:', process.env.DB_USER);
console.log('DB_NAME:', process.env.DB_NAME);
"

# 4. 完全停止并删除进程
echo ""
echo "=== 4. 完全停止并删除进程 ==="
pm2 stop life-design-backend
pm2 delete life-design-backend
pm2 save --force  # 清除保存的配置

# 5. 等待几秒确保进程完全停止
sleep 3

# 6. 重新启动
echo ""
echo "=== 5. 重新启动 ==="
pm2 start src/server.js --name life-design-backend --update-env

# 7. 等待并查看日志
sleep 5
pm2 logs life-design-backend --lines 30 --nostream
```

## 🔧 如果仍然失败：检查连接池创建时机

**问题可能是 `src/db.js` 在模块加载时就创建了连接池，但此时环境变量可能还没加载。**

**检查 `src/db.js` 和 `src/server.js` 的加载顺序：**

```bash
cd /root/apps/life-design-backend

# 检查 server.js
echo "=== server.js ==="
cat src/server.js

# 检查 db.js
echo ""
echo "=== db.js ==="
cat src/db.js

# 检查 app.js 是否导入了 db.js
echo ""
echo "=== 检查 app.js 导入 ==="
grep -n "import.*db\|from.*db" src/app.js || echo "未找到 db 导入"
```

---

## ✅ 解决方案：确保 dotenv 在连接池创建前加载

**如果 `src/db.js` 中的 `dotenv.config()` 执行太晚，需要确保在 `server.js` 中先加载：**

**修改 `src/server.js`：**

```javascript
import dotenv from "dotenv";
dotenv.config(); // 必须在最前面

import app from "./app.js";

const PORT = process.env.PORT || 3000;

app.listen(PORT, () => {
  console.log(`🚀 Backend running on port ${PORT}`);
});
```

**修改 `src/db.js`：**

```javascript
import mysql from "mysql2/promise";
// 不要在这里调用 dotenv.config()，因为 server.js 已经加载了

export const pool = mysql.createPool({
  host: process.env.DB_HOST,
  port: Number(process.env.DB_PORT),
  user: process.env.DB_USER,
  password: process.env.DB_PASS,
  database: process.env.DB_NAME,
  connectionLimit: 10,
});
```
