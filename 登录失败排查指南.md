## 🚨 常见原因

### 原因 1：验证码错误或已过期

**问题：**
- 验证码输入错误
- 验证码已过期（通常5分钟）
- 验证码已被使用

**解决：**
- 重新获取验证码
- 确认验证码正确
- 检查验证码是否过期

---

### 原因 2：数据库连接问题

**问题：**
- 后端仍连接到旧数据库
- 新数据库中没有 `sms_codes` 表
- 数据库连接配置错误

**解决：**
- 确认后端连接到新数据库 `life_design`
- 确认 `sms_codes` 表存在
- 检查数据库连接配置

---

### 原因 3：验证码表不存在

**问题：**
- 新数据库中可能没有 `sms_codes` 表

**解决：**
- 检查 `sms_codes` 表是否存在
- 如果不存在，需要创建表

---

### 原因 4：后端代码问题

**问题：**
- 登录接口代码有bug
- 验证码验证逻辑错误
- 数据库查询错误

**解决：**
- 检查后端日志错误信息
- 检查登录接口代码
- 修复代码问题

### 2. 检查 sms_codes 表是否存在

**在 DMS 中执行：**
```sql
USE life_design;
SHOW TABLES LIKE 'sms_codes';
```

**如果表不存在，需要创建：**
```sql
CREATE TABLE IF NOT EXISTS sms_codes (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  phone VARCHAR(20) NOT NULL,
  code VARCHAR(10) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  used TINYINT DEFAULT 0,
  INDEX idx_phone (phone),
  INDEX idx_code (code)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```
