# 📚 数据库迁移超详细步骤（小白版）

## 🎯 目标
在 `users` 表中添加 `password` 字段，使修改密码功能正常工作。

**数据库实例ID：** `rm-2ze0yppdih8t4t4e4`

---

## 📋 完整步骤（带截图说明）

### 步骤 1：登录阿里云控制台（2分钟）

#### 1.1 打开浏览器
- 推荐使用：Chrome、Edge、Safari
- 不要使用：IE浏览器（已不支持）

#### 1.2 访问阿里云控制台
1. **打开浏览器**
2. **在地址栏输入：** `https://ecs.console.aliyun.com`
3. **按回车键**

#### 1.3 登录账号
1. **输入你的阿里云账号**（手机号或邮箱）
2. **输入密码**
3. **点击"登录"按钮**
4. **如果开启了二次验证**（手机验证码/邮箱验证码），按提示完成验证

---

### 步骤 2：找到你的数据库实例（3分钟）

#### 2.1 进入 RDS 控制台
1. **登录成功后**，你会看到阿里云控制台首页
2. **在左侧菜单栏**（通常是深色背景），找到并点击：
   ```
   🗄️ 云数据库 RDS
   ```
   - 如果找不到，可以：
     - 在顶部搜索框输入"RDS"搜索
     - 或者直接访问：`https://rdsnext.console.aliyun.com`

#### 2.2 选择数据库实例
1. **进入 RDS 控制台后**，你会看到一个列表，显示所有数据库实例
2. **找到实例ID为：** `rm-2ze0yppdih8t4t4e4` 的实例
   - 实例ID通常在列表的第一列或第二列
   - 如果列表很长，可以使用浏览器的搜索功能（Ctrl+F 或 Cmd+F）搜索 `rm-2ze0yppdih8t4t4e4`
3. **点击这一行**（点击实例名称或任意位置都可以）

---

### 步骤 3：打开数据管理工具 DMS（2分钟）

#### 方法 A：通过 RDS 控制台（推荐）

1. **点击实例后**，你会进入实例详情页面
2. **在页面顶部或左侧**，找到以下按钮之一：
   - **"登录数据库"** 按钮（通常是蓝色或绿色）
   - **"数据管理（DMS）"** 按钮
   - **"DMS"** 按钮
3. **点击这个按钮**

#### 方法 B：直接访问 DMS（如果方法A找不到）

1. **在新标签页打开：** `https://dms.console.aliyun.com`
2. **在左侧实例列表中**，找到并点击：`rm-2ze0yppdih8t4t4e4`
3. **点击"登录"或"连接"**

---

### 步骤 4：打开 SQL 窗口（2分钟）

#### 4.1 进入 SQL 操作界面
1. **在 DMS 页面顶部**，你会看到一排菜单：
   ```
   [首页] [SQL窗口] [数据管理] [结构设计] ...
   ```
2. **点击"SQL窗口"**（或"SQL控制台"、"SQL操作"）

#### 4.2 选择数据库
1. **在 SQL 窗口左侧**，你会看到一个数据库列表（树形结构）
2. **找到并点击：** `life_design`
   - 如果看不到数据库列表，在页面顶部或左侧找一个下拉菜单
   - 选择 `life_design` 数据库
3. **确认已选中**：`life_design` 应该高亮显示或显示在顶部

---

### 步骤 5：执行 SQL 语句（3分钟）

#### 5.1 复制 SQL 语句
**复制以下全部内容**（包括分号）：

```sql
USE life_design;
ALTER TABLE users ADD COLUMN password VARCHAR(255) NULL AFTER phone;
```

#### 5.2 粘贴到 SQL 窗口
1. **在 SQL 窗口的编辑区域**（通常是白色大文本框）
2. **点击鼠标左键**，确保光标在编辑区域
3. **粘贴 SQL**：
   - Windows/Linux：按 `Ctrl + V`
   - Mac：按 `Cmd + V`
4. **确认粘贴成功**：应该能看到两行 SQL 语句

#### 5.3 执行 SQL
1. **找到执行按钮**，通常是以下之一：
   - **"执行"** 按钮（蓝色或绿色，通常在右上角）
   - **"运行"** 按钮
   - **"执行SQL"** 按钮
   - 或者按键盘快捷键：`F8`（Windows/Linux）或 `Cmd + Enter`（Mac）
2. **点击执行按钮**

#### 5.4 等待执行完成
- **执行时间：** 通常 1-5 秒
- **你会看到：**
  - ✅ **成功提示**：显示"执行成功"、"Affected rows: 1" 等
  - ❌ **错误提示**：显示红色错误信息（见下方"常见错误"部分）

---

### 步骤 6：验证字段已添加（2分钟）

#### 6.1 执行验证 SQL
1. **清空 SQL 窗口**（选中所有内容，按 Delete 或 Backspace）
2. **复制并粘贴以下 SQL：**
   ```sql
   DESCRIBE users;
   ```
3. **点击执行按钮**

#### 6.2 查看结果
**执行后，你会看到一个表格，显示 `users` 表的所有字段。**

**找到 `password` 字段：**
- 在表格中查找 `password` 这一行
- 应该能看到类似这样的信息：
  ```
  Field: password
  Type: varchar(255)
  Null: YES
  Key: (空)
  Default: NULL
  Extra: (空)
  ```

**✅ 如果看到了 `password` 字段 → 迁移成功！**

**❌ 如果没有看到 `password` 字段 → 回到步骤 5 重新执行**

## 🚨 常见错误及解决方法

### 错误 1：找不到数据库实例

**问题：** 在 RDS 列表中看不到 `rm-2ze0yppdih8t4t4e4`

**解决方法：**
1. **检查区域**：确认页面顶部显示的区域是否正确（如：华东1-杭州）
2. **检查账号**：确认登录的是正确的阿里云账号
3. **搜索功能**：使用浏览器搜索（Ctrl+F）搜索实例ID

---

### 错误 2：权限不足

**错误信息：**
```
ERROR 1142 (42000): ALTER command denied to user 'xxx'@'xxx'
```

**解决方法：**
1. **确认登录用户**：检查 DMS 中使用的数据库用户名
2. **联系管理员**：如果权限不足，需要联系数据库管理员授权
3. **使用主账号**：尝试使用阿里云主账号登录

---

### 错误 3：字段已存在

**错误信息：**
```
ERROR 1060 (42S21): Duplicate column name 'password'
```

**解决方法：**
- ✅ **这是好消息！** 说明字段已经存在
- **执行验证 SQL：** `DESCRIBE users;` 确认字段存在
- **可以忽略此错误**，直接进行验证步骤

---

### 错误 4：表不存在

**错误信息：**
```
ERROR 1146 (42S02): Table 'life_design.users' doesn't exist
```

**解决方法：**
1. **确认数据库名称**：必须是 `life_design`（不是 `life-design`）
2. **确认表名**：必须是 `users`（不是 `user`）
3. **检查数据库**：执行 `SHOW TABLES;` 查看所有表

---

### 错误 5：SQL 语法错误

**错误信息：**
```
ERROR 1064 (42000): You have an error in your SQL syntax
```

**解决方法：**
1. **检查 SQL 语句**：确保完全复制了 SQL（包括分号）
2. **检查引号**：确保使用的是英文单引号 `'`，不是中文引号 `'`
3. **重新复制粘贴**：从文档中重新复制 SQL 语句

---

### 错误 6：连接超时

**错误信息：**
```
Connection timeout
连接超时
```

**解决方法：**
1. **检查网络**：确保网络连接正常
2. **刷新页面**：按 `F5` 刷新页面
3. **重新登录**：退出 DMS，重新登录
4. **检查白名单**：确认你的 IP 地址在数据库白名单中

---
