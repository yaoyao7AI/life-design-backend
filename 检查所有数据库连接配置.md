# 🔍 检查所有数据库连接配置

## ❌ 问题

**错误日志仍然显示旧的数据库地址：**
```
Error: getaddrinfo ENOTFOUND rm-2zec076upfs3zd44110.mysql.rds.aliyuncs.com
```

**但配置文件中已经是正确的：**
```
DB_HOST=rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com
```

**说明可能有其他地方也在使用数据库配置！**

---

## 🔍 检查步骤

### 步骤 1：检查所有数据库配置文件

**在服务器上执行：**

```bash
cd /root/apps/life-design-backend

# 1. 检查所有数据库配置文件
echo "=== 检查所有数据库配置文件 ==="
find src/ -name "*.js" -exec grep -l "createPool\|mysql\|DB_HOST\|MYSQL_HOST" {} \;

# 2. 检查每个文件的内容
echo ""
echo "=== src/db.js ==="
cat src/db.js

echo ""
echo "=== src/config/database.js ==="
cat src/config/database.js 2>/dev/null || echo "文件不存在"

# 3. 检查哪些文件导入了数据库配置
echo ""
echo "=== 检查导入数据库配置的文件 ==="
grep -r "from.*db\|require.*db\|from.*database\|require.*database" src/ | grep -v node_modules
```

---

### 步骤 2：检查是否有多个数据库连接池

**在服务器上执行：**

```bash
cd /root/apps/life-design-backend

# 检查是否有多个 createPool 调用
grep -r "createPool" src/ --include="*.js"
```

---

### 步骤 3：检查 server.js 的启动顺序

**在服务器上执行：**

```bash
cd /root/apps/life-design-backend

# 检查 server.js 的完整内容
cat src/server.js
```

---

## ✅ 解决方案

### 如果发现多个数据库配置文件

**需要确保所有文件都使用正确的环境变量！**

---

### 如果发现 server.js 中 dotenv 加载顺序不对

**确保 `dotenv.config()` 在所有其他导入之前执行！**
