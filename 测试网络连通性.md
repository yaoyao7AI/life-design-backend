## 🔍 测试步骤

### 步骤 1：测试 DNS 解析

**在服务器上执行：**

```bash
# 测试 DNS 解析
echo "=== 1. 测试 DNS 解析 ==="
nslookup rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com

# 或者使用 ping（只测试 DNS，不测试端口）
ping -c 3 rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com
```

---

### 步骤 2：测试端口连通性

**在服务器上执行：**

```bash
# 测试端口 3306 是否可达
echo "=== 2. 测试端口连通性 ==="
timeout 5 bash -c "</dev/tcp/rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com/3306" && echo "✅ 端口 3306 可达" || echo "❌ 端口 3306 不可达"

# 或者使用 telnet（如果没有 telnet，使用上面的方法）
# telnet rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com 3306
```

---

### 步骤 3：使用 telnet 或 nc 测试

**如果服务器有 telnet 或 nc：**

```bash
# 使用 nc (netcat) 测试
echo "=== 3. 使用 nc 测试 ==="
nc -zv -w 5 rm-2ze0yppdih8t4t4e4.mysql.rds.aliyuncs.com 3306 && echo "✅ 连接成功" || echo "❌ 连接失败"
```

---

### 步骤 4：检查服务器私网IP

**确认服务器私网IP是否正确：**

```bash
echo "=== 4. 检查服务器网络配置 ==="
echo "服务器私网IP: $(hostname -I | awk '{print $1}')"
echo "服务器公网IP: $(curl -s ifconfig.me)"
echo ""
echo "RDS白名单应该包含："
echo "  - 服务器私网IP: 172.30.67.7"
echo "  - 或整个VPC网段: 172.16.0.0/12"
```

## 💡 可能的原因和解决方案

### 如果 DNS 解析失败：
- 检查服务器 DNS 配置
- 检查 RDS 实例是否正常运行

### 如果端口不可达：
1. **检查 RDS 白名单：**
   - 确认已添加服务器私网IP：`172.30.67.7`
   - 或确认已添加整个VPC网段：`172.16.0.0/12`

2. **检查 RDS 实例状态：**
   - 确认 RDS 实例正在运行
   - 确认内网地址正确

3. **检查 VPC 配置：**
   - 确认服务器和 RDS 在同一个 VPC
   - 检查安全组规则
